토론 역할 템플릿 설정

근거 기반 토론과 프로젝트·기획 토론에 사용할 역할 템플릿 preset을 YAML로 관리합니다. 워크벤치에서는 이 preset을 그대로 쓰거나, `Custom`으로 2~6명 구성을 직접 만들 수 있습니다.

1
템플릿 선택

근거 기반 토론과 프로젝트 워크플로우별 역할 세트를 분리해 관리합니다.

2
YAML 편집

역할 수, 역할 설명, 추천 provider와 기본 judge를 하나의 설정 파일에서 수정합니다.

설정 파일

Guide

권장 규칙

역할이 많아질수록 토큰 비용과 중복 발화가 커지므로 역할 정의를 짧고 선명하게 유지합니다.

근거 기반 토론

경제, 시장, 정책/법률 관점을 분리하고 3인 구성을 우선 추천합니다.

프로젝트·기획

기존 개선은 UX, 구현, QA를 나누고, 신규 기획은 제품 전략, 사용자 문제, MVP 검증 관점을 분리하는 편이 좋습니다.

같은 모델 재사용

가능하지만 역할 설명, 우선순위, 필수 질문을 명확히 적어야 답변이 수렴하지 않습니다.

역할 템플릿 YAML

워크플로우, 템플릿, 역할 정의를 직접 수정합니다. 저장 전에는 미리보기 카드에서 구조를 다시 확인하세요.

미리볼 템플릿이 아직 없습니다.

YAML이 로드되면 뉴스 토론용 템플릿이 이곳에 정리됩니다.

UX, 구조, 운영, 신규 기획 관점 템플릿을 비교해 기본값을 정합니다.

미리볼 템플릿이 아직 없습니다.

YAML이 로드되면 프로젝트·기획용 템플릿이 이곳에 정리됩니다.

Example

권장 템플릿 구조

역할 정의가 지나치게 길어지지 않도록, 역할 설명과 필수 질문은 한 번에 읽히는 길이로 유지합니다.

version: 1
workflows:
  news:
    templates:
      - id: market-triad
        label: 시장 영향 3자 토론
        description: 시장과 정책, 규제 리스크를 함께 본다.
        participants:
          - roleId: macro
            label: 경제 전문가
            focus: 거시경제와 산업 파급을 해석한다.
            defaultProvider: claude
            instructions:
              - 단기와 중기 영향을 나눠 설명한다.
          - roleId: equities
            label: 주식 분석가
            focus: 상장사와 투자 신호를 해석한다.
            defaultProvider: codex
          - roleId: policy
            label: 정책 관점
            focus: 집행 가능성과 운영 리스크를 본다.
            defaultProvider: gemini
  project:
    templates:
      - id: product-triad
        label: 제품 개선 3자 토론
        description: UX와 구현, 검증 관점을 함께 본다.
        participants:
          - roleId: ux
            label: UX 전문가
            focus: 사용자 흐름과 정보 구조를 점검한다.
            defaultProvider: claude
          - roleId: backend
            label: 백엔드 개발자
            focus: 구현 난이도와 운영 복잡도를 검토한다.
            defaultProvider: codex
          - roleId: qa
            label: QA
            focus: 회귀 위험과 테스트 전략을 검토한다.
            defaultProvider: gemini
